In webhookHandlers.ts, when a user completes a Stripe checkout and pays real money, the webhook handler only sends a Zapier notification — it never writes the new subscription tier to the database. usersTable.subscriptionTier is never updated. The user pays, Stripe confirms, but your app still shows them as free tier. You are taking money without delivering the product.

CHANGE 1 — checkout.session.completed handler (lines 143-152):

• If the DB write fails, throw an error so Stripe retries the webhook — do not silently swallow the failure
