SYSTEM MODE: QUANT ENGINE — MASS STRATEGY SIMULATION — NO DRIFT

OBJECTIVE:
Merge external open-source trading logic (e.g., MiroFishes GitHub) into a custom simulation engine that runs 10,000 trading strategies across live and historical stock data, ranks their performance, and returns the top 100 trading opportunities to users.

---

1. CORE SYSTEM DESIGN

Build a Strategy Simulation Engine where:
- Each “agent” = 1 independent trading strategy
- Total strategies = scalable up to 10,000
- All strategies run in parallel or batched execution

---

2. STRATEGY STRUCTURE

Each strategy must follow this interface:

strategy({
  price_data,
  volume_data,
  indicators,
  config
}) → {
  signals: [BUY, SELL, HOLD],
  confidence: 0–1,
  expected_return,
  risk_score
}

---

3. DATA INPUT

- Use real-time + historical stock data
- Must support:
  - OHLC price data
  - volume
  - indicators (RSI, MACD, EMA, etc.)

- No fake or simulated market data allowed

---

4. STRATEGY TYPES (GENERATE MANY VARIATIONS)

Programmatically generate strategies using:
- Parameter variations:
    RSI(14), RSI(7), RSI(21)
    EMA(9/21), EMA(20/50), EMA(50/200)
- Combine indicators:
    RSI + MACD
    EMA + Volume
- Randomized configs within valid bounds

---

5. EXECUTION ENGINE

- Run all strategies across selected stocks
- Use batching or worker threads (Node workers / Python multiprocessing)
- Optimize for speed (parallel execution)

---

6. SCORING SYSTEM (CRITICAL)

Each strategy output must be scored using:

score = weighted(
  expected_return,
  win_rate,
  drawdown,
  volatility,
  confidence
)

---

7. RANKING SYSTEM

- Rank all strategy results
- Select:
    top 10,000 strategies → filter → top 100 signals

- Deduplicate:
    avoid repeating same stock excessively

---

8. OUTPUT FORMAT

Return top 100 opportunities:

[
  {
    symbol: "AAPL",
    action: "BUY",
    confidence: 0.92,
    expected_return: 8.5%,
    risk_score: low,
    strategy_id: "EMA_RSI_21_9"
  }
]

---

9. INTEGRATION WITH YOUR APP

- Feed results into:
    dashboard
    alerts
    trading panel

- Allow users to:
    click → trade instantly

---

10. PERFORMANCE REQUIREMENTS

- Must handle:
    thousands of strategies × hundreds of stocks
- Use:
    caching
    batching
    async execution

---

11. LOGGING + DEBUG

- Track:
    strategy performance
    failures
    execution time

---

FINAL RULES:

- No fake predictions
- No hallucinated signals
- All outputs must be derived from actual data + deterministic logic
- System must be reproducible and testable
