# CORE DESIGN PHILOSOPHY

1. CLARITY OVER COMPLEXITY
If a user cannot understand the screen in 3 seconds → it fails

2. LESS BUT BETTER
Remove anything unnecessary

3. VISUAL HIERARCHY IS EVERYTHING
User must instantly know:
- what matters most
- what to do next

4. EVERY PIXEL HAS A PURPOSE
No decorative nonsense

---

# TASK

Audit and redesign the current UI.

You must:

1. REMOVE:
- redundant elements
- clutter
- confusing sections
- duplicate actions

2. SIMPLIFY:
- navigation
- layouts
- text

3. REBUILD:
- clean layout structure
- proper spacing
- strong hierarchy

---

# LAYOUT RULES

- Max 3 primary sections per screen
- One clear primary action per view
- Use whitespace aggressively
- Group related elements visually

---

# TYPOGRAPHY

- Large, bold headline (clear purpose)
- Medium subtext (supporting)
- Small detail text (secondary only)

No tiny unreadable garbage.

---

# COLOR SYSTEM

- 1 primary color
- 1 accent color
- neutrals for everything else

Use color ONLY to:
- guide attention
- signal importance

---

# COMPONENT RULES

Buttons:
- clear label
- strong contrast
- obvious action

Cards:
- minimal content
- clear purpose

Forms:
- short
- simple
- no overload

---

# USER FLOW (MANDATORY)

User must:

1. Land
→ instantly understand what this is

2. See value
→ within 5 seconds

3. Take action
→ with zero confusion

---

# DESIGN TEST

Before finalizing ANY screen:

Ask:

- Can a 12-year-old understand this?
- Is anything unnecessary?
- Is the main action obvious?
- Does this feel calm or chaotic?

If chaotic → redesign

---

3. 🟡 What is unnecessary
4. 🟢 What should remain
5. 🛠 Exact redesign (layout + structure)
